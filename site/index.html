SCRUM MCP turns multi-agent chaos into a dependable dev team.

SCRUM (Synchronised Claims Registry for Unified Multi-agents) is the coordination layer that keeps every agent honest and in sync. Declare intent, claim files, attach evidence, and stream realtime signals without phoning home.

New in v0.2.0

Full Kanban workflow for multi-agent teams.

v0.2.0 transforms SCRUM from file coordination into a complete project management system for agents. Agents track tasks through Kanban columns, coordinate dependencies, respect WIP limits, and report velocity metrics autonomously.

Kanban Board

Visual task workflow with Backlog, To Do, In Progress, Review, and Done columns. Assign agents, set priorities, track due dates.

Comments & Blockers

Discuss tasks with comments. Track blockers and impediments. Blocked tasks cannot move forward until resolved.

Dependencies

Define task dependencies. Check if tasks are ready to start. Prevents circular dependencies and enforces ordering.

WIP Limits

Set work-in-progress limits per column. Prevent overloading. Visual indicators when limits are exceeded.

Metrics Dashboard

Track cycle time, lead time, and throughput. Measure velocity. Identify aging tasks stuck in progress.

Agent Registry

Register agents for observability. Track heartbeats and status. See who's active, idle, or offline. Find abandoned work.

Approval Gates

Define lint, test, and build gates that must pass before status transitions. Record gate runs with output. Block on failing gates.

Task Templates

Create reusable task patterns with placeholders. Pre-configure gates, labels, priority, and acceptance checklists.

Webhooks

Push events to external systems. HMAC signing for security. Track delivery history for debugging.

The coordination surface

Every agent gets the same truth, in realtime.

SCRUM MCP is not another dashboard. It is the protocol level source of truth that keeps intent, ownership, and evidence from drifting apart.

Intent announcements

Declare what you plan to change before edits happen. SCRUM records it and broadcasts the signal.

File claims

Short-lived file ownership prevents collisions. No more surprise merges at 2AM.

Overlap detection

Catch conflicts before they explode. Agents know who is touching what in real time.

Evidence receipts

Attach command output as proof. If it is not evidenced, it did not happen.

Realtime events

Stream updates to every connected tool with WebSocket events and a single source of truth.

Local-first trust

SCRUM never phones home. It runs where your repo lives and never edits without consent.
