SCRUM MCP turns multi-agent chaos into a dependable dev team.

SCRUM (Synchronised Claims Registry for Unified Multi-agents) is the coordination layer that keeps every agent honest and in sync. Declare intent, claim files, attach evidence, and stream realtime signals without phoning home.

New in v0.5.1

Smart tool consolidation for better usability.

v0.5.1 reduces MCP tools from 56 to 32 through smart consolidation. Action-based tools (blocker, dependency) and type-based tools (metrics) reduce cognitive load. Profile-based discovery shows only what you need. Admin tools moved to REST-only for cleaner agent experience.

56 → 32 Tools

Smart consolidation reduces cognitive load by 43%. Action-based and type-based patterns replace redundant individual tools.

Profile Discovery

scrum_status(profile: "solo|team|full") shows only relevant tools. Solo: 13 core. Team: 22. Full: all 32.

Unified Blocker

scrum_blocker(action: "add|resolve|list") replaces 3 separate tools. One tool, clear actions.

Unified Dependency

scrum_dependency(action: "add|remove|get|check") replaces 4 tools. Includes task readiness check.

Unified Metrics

scrum_metrics(type: "board|velocity|aging|task") replaces 4 tools. All metrics in one place.

REST-Only Admin

WIP limits, gates, templates, webhooks, agent registry moved to REST API. Cleaner MCP surface for agents.

Sprint Collaboration

Create sprints where multiple agents coordinate on the same task. Share decisions, interfaces, Q&A.

Compliance Check

Verify agents did what they declared. Block releases on undeclared files or boundary violations.

Kanban + Metrics

Full board with dependencies, WIP limits, blockers. Track cycle time, velocity, aging WIP.

Schema Validation

Defensive constraints on all fields. TaskIdField, AgentIdField ensure proper input validation.
